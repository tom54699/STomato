# UI 原型狀態追蹤

最後更新：2025-12-03

## 畫面清單與狀態

### 1. Login（登入頁）
- **完成度**: 100%
- **功能**:
  - 使用者登入
  - LocalStorage 儲存登入狀態
- **缺口**: 無後端整合、無真實驗證機制
- **下一步**: 待後端 API 就緒後整合真實登入流程

### 2. Home（首頁/番茄鐘主畫面）
- **完成度**: 95%
- **功能**:
  - 圓環拖曳調整番茄鐘時間 (5-120 分鐘)
  - 開始/暫停/繼續計時器
  - 今日計畫列表與快速啟動
  - **累積番茄鐘追蹤**：顯示計畫進度（已完成/目標分鐘數、番茄鐘數量）
  - **計畫按鈕重新設計**：圓形圖示按鈕，顯示圓環設定的分鐘數
  - **自動完成**：達到目標時長時自動標記計畫完成
  - 結算頁面跳轉
- **缺口**:
  - 無通知權限請求
  - 無音效/震動提醒
- **下一步**:
  - 新增瀏覽器通知 API 整合
  - 新增完成音效

### 3. StudyPlanner（讀書計畫）
- **完成度**: 95%
- **功能**:
  - **下拉選單設計**：藍色漸層（學習時長）、綠色漸層（開始時間）
  - **時間衝突檢測**：不可用時段標示（❌ 與 XXX 計畫衝突）
  - **累積追蹤欄位**：targetMinutes、completedMinutes、pomodoroCount
  - 15 分鐘間隔時間選擇（7:00-22:00）
  - 計畫列表：顯示今日、明日、已完成計畫
  - 編輯/刪除功能
  - LocalStorage 持久化
- **缺口**:
  - 無行事曆視圖
  - 無跨日計畫
  - 無重複計畫功能
- **下一步**:
  - 新增週檢視/月檢視行事曆
  - 支援計畫複製與範本功能
- **Mock 資料**: 參見 notes/2025-12-03-claude.md

### 4. Schedule（時程表）
- **完成度**: 80%

## 整體進度總結

- **核心功能完成度**: 85%
- **主要完成**：
  - 番茄鐘計時器與累積追蹤系統
  - 讀書計畫 CRUD 與衝突檢測
  - 下拉選單 UI 改進（漸層設計、emoji 標示）
  - 圓環調整與計畫按鈕同步
  - 基本統計與歷史記錄
  - LocalStorage 資料持久化

- **主要缺口**：
  - 後端 API 整合（登入、排行榜、社交功能）
  - 通知與音效系統
  - 進階篩選與搜尋
  - 成就與徽章系統

- **技術債**：
  - Mock 資料分散在各元件中，需統一管理
  - LocalStorage 使用較多，需考慮資料遷移策略
  - 部分元件耦合度較高，可重構提取共用邏輯
