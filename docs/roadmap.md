# Roadmap

## 近期（React 原型）
- 確認 MVP 流程與畫面，標記已完成與缺口。
- 產生模擬資料/假 API 以驗證互動。

## 中期（Flutter 移植）
- 將確認過的流程/元件翻譯成 Flutter 小部件與路由。
- 決定狀態管理（例：Riverpod/Bloc）與主題系統。

## 後端
- 認證與使用者模型。
- 計時/積分/排行榜/進度資料的 API 與儲存。
- 事件/挑戰/課程內容的結構化定義。

## 里程碑示例
- M0：React 原型的核心畫面互動完成（登入、排程、計時、積分/進度檢視、社交挑戰）。
- M1：凍結 UI/UX，輸出 Flutter/後端需求列表與介面契約。
- M2：Flutter MVP + 後端最小 API。
